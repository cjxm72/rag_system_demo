# 论文写作素材（可直接改写入论文）

本文件聚焦论文写作可用的技术描述，避免与实现细节重复。

## 1. 系统总体架构（摘要版）

系统由前端静态页面与后端 FastAPI 服务组成：

- **前端**：对话交互、文档上传、知识组选择、配置管理（API Key/模型等保存在 LocalStorage，每次请求随 `settings` 发送）。
- **后端**：文档解析（PDF 优先 OpenDataLoader v2+，回退 PyMuPDF；DOCX 用 python-docx；图片走多模态 Chat）、Milvus 向量索引构建、检索加权与（可选）rerank、多智能体路由与生成、PostgreSQL 引用反查、评估接口。

## 2. 知识组织（Milvus + PostgreSQL）

- **向量索引（Milvus）**：文档切 chunk 后写入单 collection；字段含 `doc_id`、`chunk_index` 等，便于检索后反查业务表。
- **PostgreSQL（SQLModel）**：存储文档全文与文件名、知识组与映射、对话记忆；引用阶段用 `doc_id` 回查文档名，提高可解释性。
- **知识组**：仅存于 PostgreSQL，不向向量库写入 `group_id`；查询时以 doc_id 集合过滤向量召回结果。

## 3. 检索与排序（当前实现）

1. 向量召回：问题向量化，在 Milvus 召回 topK 候选 chunk；
2. 组过滤：用 `doc_id in (...)` 在 Milvus 侧过滤候选；
3. 组权重加权：对候选分数做加权，体现主/次组优先级；
4. （可选）在线 rerank：若配置硅基 rerank 则调用 `/rerank` 精排，否则按向量/权重排序截断；
5. 生成：将上下文送入 LLM，文末由系统按 doc_id 回查并追加引用列表。

## 4. 多智能体路由（协调者自主决策）

路由策略由**协调者模型**根据用户问题输出结构化 JSON（domain + delegates + reason），自主决定是否调用医疗/法律专家模型或通过 Team coordinate 融合输出，并输出 `【路由说明】...` 便于实验记录。
