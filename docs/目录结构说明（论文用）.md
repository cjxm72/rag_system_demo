# 目录结构说明（论文用）

本文档用于论文写作：解释项目目录结构、各文件职责，以及给出 PostgreSQL 建表 SQL（含索引）方便迁移。

## 1. 顶层目录

- `rag_demo/`：后端核心包（FastAPI + RAG + 存储 + 解析 + 多智能体）
- `static/`：前端静态页（问答界面 + 设置页）
- `tests/`：pytest 集成测试（端到端 + 评估）
- `docs/`：技术文档（中文命名）
- `.env.example`：本机 PostgreSQL/Milvus 默认配置示例
- `pyproject.toml` / `uv.lock`：依赖与锁文件

## 2. 后端包 `rag_demo/`

### 2.1 `rag_demo/api/`（接口层）

- `rag_demo/api/main.py`
  - 静态页：`GET /`、`GET /settings`
  - 问答：`POST /query`（**SSE：text/event-stream**，唯一问答出口）
  - 文档：`GET /documents`、`POST /documents/upload`、`DELETE /documents/{id}`
  - 知识组：`GET /groups`、`POST /groups`、`PUT /groups/{id}`、`DELETE /groups/{id}`
  - 评估：`POST /evaluate`
- `rag_demo/api/sse.py`：SSE 帧编码（`data: {json}\n\n`）

### 2.2 `rag_demo/core/`（基础设施）

- `rag_demo/core/provider_settings.py`
  - 将前端 settings 解析为统一的 OpenAI 兼容 endpoint
  - 支持：**硅基 / OpenAI / Ollama**
  - 允许 Embedding / Rerank / LLM / Vision 各自独立选择提供商与专用 base/key
- `rag_demo/core/singleton.py`：`@singleton` 缓存模型/Team 构建，降低重复初始化开销

### 2.3 `rag_demo/storage/`（PostgreSQL + SQLModel）

- `rag_demo/storage/models.py`：SQLModel 表结构 + 索引定义
- `rag_demo/storage/database.py`：读取 `.env`，创建 Engine，设置 `search_path=public`
- `rag_demo/storage/db.py`：业务 CRUD（documents/groups/chat_messages/app_meta）
- `rag_demo/storage/store.py`：对上层导出与 `db.py` 相同的 API

### 2.4 `rag_demo/rag/`（RAG 核心）

- `rag_demo/rag/rag_system.py`
  - 文档切块：`SentenceSplitter(chunk_size, chunk_overlap)`
  - 向量化：OpenAI 兼容 `/embeddings`
  - 检索：Milvus 相似度召回 + doc_id 过滤表达式
  - 排序：知识组权重加权 +（可选）硅基 `/rerank`
  - 引用：doc_id 反查文档名，追加 `【引用文档】`
- `rag_demo/rag/milvus_store.py`：pymilvus 连接与 collection 操作封装
- `rag_demo/rag/siliconflow_embedding.py`：OpenAI 兼容 embedding 调用实现
- `rag_demo/rag/siliconflow_rerank.py`：硅基风格 `/rerank` 调用实现
- `rag_demo/rag/types.py`：RAG 返回结构（chunks / citation_doc_ids）

### 2.5 `rag_demo/parsing/`（文档解析）

- `rag_demo/parsing/doc_parser.py`
  - PDF：OpenDataLoader v2+（失败回退 PyMuPDF）
  - DOCX：python-docx
  - 图片：调用 `vision_api.py` 描述图片文本/内容
- `rag_demo/parsing/vision_api.py`：OpenAI 兼容多模态 Chat（硅基/OpenAI/Ollama）

### 2.6 `rag_demo/agents/`（多智能体）

- `rag_demo/agents/agno_team.py`
  - **协调者模型**做路由：输出 JSON（domain + delegates + reason）
  - 按 delegates 决定调用 Medical / Legal 或 Team coordinate 融合输出

## 3. 前端 `static/`

- `static/index.html`
  - 对话界面：通过 SSE 调用 `POST /query`，按 `data: {...}` 事件逐步渲染
  - 右侧：文档上传、知识组管理
  - 含“论文段落”便于复制（与 `docs/论文修改及流程图生成.md` 一致口径）
- `static/settings.html`
  - 多提供商配置：Embedding / Rerank / LLM / Vision 分别配置 provider/base/key/model
  - 保存到 LocalStorage（key: `rag_settings`）

## 4. 测试 `tests/`

- `tests/test_e2e_rag.py`：端到端（上传→建组→SSE /query→路由/引用断言）
- `tests/test_rag_evaluate.py`：评估接口 `/evaluate` 的结构断言
- `tests/text/*.txt`：固定测试数据（法律/医疗/混合）
- `tests/conftest.py`：隔离环境、打印数据文件列表、可选打印 SSE 聚合正文

## 5. PostgreSQL 建表 SQL（含索引）

说明：
- schema：`public`
- 兼容本项目字段含义（SQLModel 版本）
- 可直接用于迁移/初始化

```sql
-- documents：文档全文与元数据
CREATE TABLE IF NOT EXISTS public.documents (
  id         varchar(64) PRIMARY KEY,
  name       text NOT NULL DEFAULT '',
  path       text NULL,
  text       text NOT NULL DEFAULT '',
  status     varchar(32) NOT NULL DEFAULT 'queued',
  progress   integer NOT NULL DEFAULT 0,
  error      text NOT NULL DEFAULT '',
  created_at timestamptz NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS ix_documents_status     ON public.documents (status);
CREATE INDEX IF NOT EXISTS ix_documents_created_at ON public.documents (created_at);

-- groups：知识组
CREATE TABLE IF NOT EXISTS public.groups (
  id   varchar(64) PRIMARY KEY,
  name text NOT NULL DEFAULT ''
);

CREATE INDEX IF NOT EXISTS ix_groups_name ON public.groups (name);

-- group_members：组-文档多对多
CREATE TABLE IF NOT EXISTS public.group_members (
  group_id varchar(64) NOT NULL,
  doc_id   varchar(64) NOT NULL,
  PRIMARY KEY (group_id, doc_id),
  CONSTRAINT fk_group_members_group
    FOREIGN KEY (group_id) REFERENCES public.groups(id) ON DELETE CASCADE,
  CONSTRAINT fk_group_members_doc
    FOREIGN KEY (doc_id) REFERENCES public.documents(id) ON DELETE CASCADE
);

CREATE INDEX IF NOT EXISTS ix_group_members_group_id ON public.group_members (group_id);
CREATE INDEX IF NOT EXISTS ix_group_members_doc_id   ON public.group_members (doc_id);

-- chat_messages：服务端会话记忆
CREATE TABLE IF NOT EXISTS public.chat_messages (
  id         bigserial PRIMARY KEY,
  thread_id  varchar(256) NOT NULL,
  role       varchar(32) NOT NULL,
  content    text NOT NULL,
  created_at integer NULL
);

CREATE INDEX IF NOT EXISTS ix_chat_messages_thread_id    ON public.chat_messages (thread_id);
CREATE INDEX IF NOT EXISTS ix_chat_messages_thread_id_id ON public.chat_messages (thread_id, id);

-- app_meta：系统元数据（例如 embedding_model / milvus_built_embedding）
CREATE TABLE IF NOT EXISTS public.app_meta (
  key   varchar(128) PRIMARY KEY,
  value text NOT NULL
);
```

## 6. 索引选择理由（论文可写）

- `documents(status)`：文档列表/进度轮询（queued/parsing/embedding/done/error）
- `documents(created_at)`：按时间展示/清理旧文档
- `group_members(group_id)`：按组取 doc_id（检索前过滤）
- `group_members(doc_id)`：删除文档时快速清理关联
- `chat_messages(thread_id, id)`：按 thread_id 取最近 N 条历史，且按 id 倒序/正序组合
