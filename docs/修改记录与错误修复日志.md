# 修改记录与错误修复日志（毕设用）

本文件记录重构过程的关键改动与修复点，便于论文与答辩说明。

## 2026-04 — 存储与输出形态升级（当前版本）

- **向量库**：Milvus（本机部署，默认 collection `rag_kb`）
- **业务库**：PostgreSQL + SQLModel（schema `public`，读取 `.env`）
- **问答输出**：仅保留 `POST /query` 的 **SSE（text/event-stream）**
- **多提供商**：Embedding / Rerank / LLM / 视觉 均支持硅基 / OpenAI / Ollama（OpenAI 兼容 HTTP）
- **路由策略**：由协调者模型输出 JSON 自主决定 delegates（不使用关键词匹配）

## 已记录的关键修复

- **PostgreSQL text 不允许 NUL**：入库前剔除 `\\x00`，避免二进制误解析导致入库失败。
- **SQLModel 查询返回 Row**：对 `select(Model)` 使用 `session.scalars(...)` 获取模型对象，避免 `AttributeError: id`。
