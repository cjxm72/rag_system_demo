# 评估说明（论文用明细指标）

本项目提供 `POST /evaluate` 用于对 RAG 的 **检索 + 生成** 进行评估，并返回可直接用于论文写作的明细数据（chunks、分数、引用、耗时、字符统计等）。

## 1. 输入

- `items[]`
  - `question`：问题
  - `expected`：期望答案（用于语义相似度 proxy）
  - `groups`：知识组选择（同 `/query` 的 `groups`）
- `settings`：与 `/query` 一致（来自前端 LocalStorage）

## 2. 输出字段（论文关键）

每个 `items[i]`：

- `answer`：LLM 实际回答
- `context`：检索上下文（含 `[来源i] doc_id=... chunk=...`）
- `semantic_similarity`：`cos(answer_embedding, expected_embedding)`

### 2.1 检索明细 `items[i].retrieval`

- `similarity_top_k`：向量召回数量
- `rerank_top_n`：重排保留数量
- `citation_doc_ids`：本轮引用涉及的 doc_id
- `chunks[]`：
  - `source_id`：上下文编号
  - `doc_id`：文档 id
  - `chunk_index`：文档内 chunk 序号
  - `score`：分数（Milvus 距离+组权重+可选 rerank）
  - `text`：chunk 原文（可用于论文附录/案例分析）

### 2.2 耗时与统计

- `timing_s`：`retrieve / llm / embeddings_for_metrics / total`
- `stats`：上下文/回答/期望长度、chunk 数、引用文档数等
