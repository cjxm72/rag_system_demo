# 架构与实现说明（当前版本）

本文档说明：总体架构、PostgreSQL（SQLModel）与 Milvus、检索与引用、对话记忆、PDF 解析、Agno Team 与接口。

## 1. 目录结构（核心代码）

| 路径 | 职责 |
|------|------|
| `rag_demo/api/main.py` | FastAPI：上传、知识组、`POST /query`（SSE）、评估 |
| `rag_demo/api/sse.py` | SSE 帧编码 |
| `rag_demo/core/provider_settings.py` | 硅基 / OpenAI / Ollama 的 Base、Key 解析 |
| `rag_demo/storage/models.py` | SQLModel 表：`Document`、`Group`、`GroupMember`、`ChatMessage`、`AppMeta` |
| `rag_demo/storage/database.py` | PostgreSQL 引擎、`.env` 加载、`create_db_and_tables` |
| `rag_demo/storage/db.py` | 业务 CRUD（PostgreSQL） |
| `rag_demo/rag/rag_system.py` | 切块、Embedding、Milvus 检索、（可选）rerank、上下文与引用 |
| `rag_demo/rag/milvus_store.py` | Milvus collection `rag_kb` 封装 |
| `rag_demo/parsing/doc_parser.py` | PDF（OpenDataLoader v2+）/ DOCX / 图片 / 文本 |
| `rag_demo/agents/agno_team.py` | 协调者路由 + 医疗/法律专家 + Team coordinate |
| `static/` | 前端静态页 |

## 2. PostgreSQL 表设计

- **documents**：`id, name, path, text, status, progress, error, created_at`
- **groups** + **group_members**：知识组与多对多文档关系
- **chat_messages**：`thread_id, role, content, created_at`
- **app_meta**：如 `embedding_model`、`milvus_built_embedding`

表由 **SQLModel** 定义，启动时 `create_all` 自动建表（schema `public`）。

## 3. 向量库（Milvus）

- 单 collection（默认 `rag_kb`），字段：`pk, doc_id, chunk_index, name, text, embedding`
- 文档变更或 Embedding 模型变更会触发 drop 并重建 collection（保证一致性）

## 4. 模型接入（硅基 / OpenAI / Ollama）

- **Embedding / 视觉 / LLM**：均为 OpenAI 兼容 HTTP（`base_url` + `api_key`）
- **Rerank**：仅硅基风格 `/rerank`（或兼容网关）启用；OpenAI/Ollama 无标准 rerank 时按向量/权重排序截断

## 5. 检索与引用

1. Milvus 相似度召回 → doc_id 过滤 → 组权重加权 →（可选）硅基 rerank
2. 上下文每条片段带 `[来源i] doc_id=... chunk=...`
3. 回答结束后按引用 doc_id 回查 PostgreSQL，追加 `【引用文档】` 列表
4. `POST /query` 以 SSE 输出：`route/chunk/sources/done`

## 6. 多智能体路由（协调者自主决策）

协调者模型基于用户问题输出 JSON（domain + delegates + reason），自主决定是否调用 Medical / Legal 专家或 Team coordinate 融合输出，并在回答中输出 `【路由说明】...`。
